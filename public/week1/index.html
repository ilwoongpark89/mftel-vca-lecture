토큰화 — 언어 모델이 글을 자르는 법

Tokenization — How a Language Model Cuts Text

🧩
직관 — LLM은 글자나 단어를 그대로 보지 않는다. 텍스트를 토큰(token)이라는 조각으로 쪼개 정수 ID로 바꾼 뒤 처리한다. 조각의 크기는 얼마나 자주 등장하느냐로 정해진다 — 흔한 덩어리는 통째로 한 토큰, 드문 덩어리는 여러 토큰으로 나뉜다. 그래서 같은 문장이라도 언어에 따라 토큰 수가 크게 달라지고, 토큰 수 = 비용이자 컨텍스트 한도가 된다. (출처: Sennrich, Haddow & Birch 2016, Neural Machine Translation of Rare Words with Subword Units, ACL / OpenAI tiktoken 문서)
🧩
Intuition — an LLM does not read characters or words directly. It splits text into pieces called tokens, maps each to an integer ID, and processes those. The size of a piece is decided by how often it appears — a frequent chunk becomes a single token, a rare chunk is split into several. So the same sentence differs sharply in token count across languages, and token count = both cost and context budget. (Source: Sennrich, Haddow & Birch 2016, Neural Machine Translation of Rare Words with Subword Units, ACL / OpenAI tiktoken docs)

서브워드 토큰화(subword tokenization)는 단어(word)와 문자(character)의 중간 단위로 텍스트를 자릅니다. 단어 단위는 미등록어(OOV)에 취약하고 문자 단위는 시퀀스가 너무 길어집니다. BPE(Byte-Pair Encoding)는 이 둘 사이에서, 코퍼스에서 가장 자주 인접한 기호 쌍을 반복적으로 하나로 병합해 어휘(vocabulary)를 만듭니다. 병합 규칙의 개수(어휘 크기)가 조각의 평균 크기를 결정합니다.

글자 수가 같아도 토큰 수는 언어·기호에 따라 크게 다릅니다. 영어는 흔한 병합이 많아 토큰이 적고, 한글·희귀 유니코드·코드 기호는 병합이 드물어 바이트 단위로 잘게 쪼개져 토큰이 많아집니다.

Subword tokenization cuts text at a level between words and characters. Word-level units are fragile to out-of-vocabulary (OOV) items, while character-level units make sequences too long. BPE (Byte-Pair Encoding) sits between them: it repeatedly merges the most frequent adjacent symbol pair in a corpus to build a vocabulary. The number of merge rules (vocabulary size) sets the average piece size.

BPE 학습 — 최빈 쌍을 반복 병합하기

Learning BPE — Iteratively Merging the Most Frequent Pair

공통 관점: BPE는 그리디 알고리즘입니다. 코퍼스를 문자(또는 바이트)로 시작해, 매 단계마다 가장 자주 붙어 나타나는 인접 기호 쌍을 골라 하나의 새 기호로 합칩니다. 이 병합을 $k$번 반복하면 어휘 크기가 $\approx |{\text{초기 기호}}| + k$가 됩니다. 흔한 단어일수록 빨리 통째 토큰이 되고, 드문 단어는 조각으로 남습니다.

Unifying view: BPE is a greedy algorithm. It starts the corpus as characters (or bytes) and, at each step, picks the adjacent symbol pair that co-occurs most often and fuses it into one new symbol. Repeating this merge $k$ times yields a vocabulary of size $\approx |{\text{initial symbols}}| + k$. Frequent words become whole tokens quickly; rare words stay as fragments.

Step 1 · 초기화

문자/바이트로 분해
🔤 "low", "lower" → l·o·w · l·o·w·e·r
$ V_0 = \{\text{개별 문자 또는 256 바이트}\} $

모든 단어를 기본 기호 시퀀스로 쪼갭니다. 바이트 단위면 어떤 유니코드도 표현 가능 — OOV 없음.

Step 2 · 최빈 쌍 선택

전 코퍼스 빈도 집계
📊 가장 자주 붙는 쌍 argmax
$ (a,b)^{*} = \underset{(a,b)}{\arg\max}\ \text{freq}(a,b) $

인접한 두 기호의 등장 횟수를 세어 최빈 쌍 $(a,b)^{*}$를 찾습니다. 예: $(l,o)$ 또는 $(o,w)$.

Step 3 · 병합 & 반복

$k$회 반복
🔗 $(a,b)^{*}\!\to\!ab$ 새 기호 추가
$ V_{i+1}=V_i \cup \{ab\},\quad |V_k|\approx|V_0|+k $

선택된 쌍을 하나로 합쳐 어휘에 추가하고 다시 Step 2로. 목표 어휘 크기까지 병합 규칙 리스트가 쌓입니다.

Step 1 · initialize

split into chars/bytes
🔤 "low", "lower" → l·o·w · l·o·w·e·r
$ V_0 = \{\text{individual chars or 256 bytes}\} $

Break every word into base-symbol sequences. Byte-level can represent any Unicode — no OOV.

Step 2 · pick top pair

count over whole corpus
📊 argmax of co-occurring pair
$ (a,b)^{*} = \underset{(a,b)}{\arg\max}\ \text{freq}(a,b) $

Count occurrences of each adjacent pair and find the most frequent pair $(a,b)^{*}$, e.g. $(l,o)$ or $(o,w)$.

Step 3 · merge & repeat

repeat $k$ times
🔗 add new symbol $(a,b)^{*}\!\to\!ab$
$ V_{i+1}=V_i \cup \{ab\},\quad |V_k|\approx|V_0|+k $

Fuse the chosen pair, add it to the vocabulary, and return to Step 2. A list of merge rules accumulates up to the target vocabulary size.

주의 · 왜 한글·코드가 토큰을 더 쓰나
BPE 병합 규칙은 학습 코퍼스의 빈도에서 나옵니다. 영어 중심 코퍼스에서는 한글 음절·희귀 기호가 자주 인접하지 못해 병합이 거의 안 되고, 결국 UTF-8 바이트(한글 1자 = 3바이트)까지 쪼개져 영어보다 훨씬 많은 토큰을 씁니다. 코드의 들여쓰기·특수기호도 마찬가지입니다. (출처: OpenAI tiktoken 문서 / Sennrich 2016)
Caution · why Korean & code cost more tokens
BPE merge rules come from the frequency in the training corpus. In an English-dominant corpus, Korean syllables and rare symbols rarely co-occur, so few merges apply and they are split down to UTF-8 bytes (one Korean syllable = 3 bytes), using far more tokens than English. The same holds for code indentation and special symbols. (Source: OpenAI tiktoken docs / Sennrich 2016)

Sennrich의 서브워드와 바이트 단위 BPE

Sennrich's Subwords and Byte-Level BPE

2016년 Rico Sennrich 등은 기계번역에서 희귀어·미등록어 문제를 풀기 위해 데이터 압축 기법 BPE를 어휘 구축에 도입했습니다(Sennrich et al. 2016, ACL). 핵심 아이디어는 단순합니다 — 자주 붙는 기호 쌍을 반복 병합하면, 흔한 단어는 통째 토큰이 되고 드문 단어는 의미 있는 조각으로 남아 OOV가 사라진다는 것입니다.

토큰의 크기는 문법이 아니라 코퍼스의 빈도가 정한다.

이후 GPT-2는 유니코드 문자가 아닌 바이트(0~255)에서 BPE를 시작하는 바이트 단위 BPE를 채택했습니다(Radford et al. 2019). 256개 바이트만 기본 기호로 두면 세상의 어떤 문자열도 표현할 수 있어 미등록어가 원천 봉쇄됩니다. 대가는, 영어 코퍼스로 학습된 병합이 한글·이모지·코드 기호에는 잘 적용되지 않아 이들이 바이트 단위로 잘게 쪼개진다는 점입니다.

꼭 알아야 할 점
  • BPE는 빈도 기반 그리디 병합 — 문법 규칙이 아니다.
  • 바이트 단위 BPE는 OOV를 구조적으로 제거한다.
  • 같은 글자 수라도 언어별 토큰 수는 다르다 → 비용·컨텍스트 차이.

참고: Sennrich et al. (2016) ACL / Radford et al. (2019) GPT-2 / OpenAI tiktoken 문서

In 2016, Rico Sennrich and colleagues introduced BPE — a data-compression method — into vocabulary building to solve the rare-word / OOV problem in machine translation (Sennrich et al. 2016, ACL). The idea is simple: repeatedly merge frequently adjacent symbol pairs so that common words become whole tokens while rare words remain meaningful fragments, and OOV disappears.

Token size is set not by grammar but by corpus frequency.

Later, GPT-2 adopted byte-level BPE, starting BPE from bytes (0–255) rather than Unicode characters (Radford et al. 2019). With only 256 base bytes, any string on Earth can be represented, closing off OOV at the source. The cost: merges learned from an English corpus apply poorly to Korean, emoji, and code symbols, so these get split down to bytes.

Must know
  • BPE is a frequency-based greedy merge, not a grammar rule.
  • Byte-level BPE structurally removes OOV.
  • Even at equal character counts, token counts differ by language → cost & context differ.

Refs: Sennrich et al. (2016) ACL / Radford et al. (2019) GPT-2 / OpenAI tiktoken docs

셀프 어텐션 — 모든 토큰이 서로를 바라본다

Self-Attention — Every Token Looks at Every Other

👀
직관 — "그 고양이가 배고파서 그것이 울었다"에서 "그것"이 누구인지 알려면, 문장 속 다른 모든 단어를 훑어 "고양이"에 가장 큰 주의(attention)를 준다. 트랜스포머는 이 "누구를 얼마나 볼지"를 학습된 가중치로 병렬 계산한다 — 각 토큰이 질의(Q)를 던지고, 다른 토큰의 열쇠(K)와의 유사도로 값(V)을 가중합한다. (출처: Vaswani et al. 2017, "Attention Is All You Need", NeurIPS)
👀
Intuition — In "the cat was hungry so it cried," to resolve what "it" refers to you scan every other word and give the largest attention to "cat." A Transformer computes this "who to look at, how much" in parallel using learned weights — each token issues a query (Q), and by its similarity to the keys (K) of other tokens it takes a weighted sum of their values (V). (Source: Vaswani et al. 2017, "Attention Is All You Need", NeurIPS)

셀프 어텐션(self-attention)은 한 시퀀스 안의 모든 토큰이 서로에게 주의를 배분해 문맥 표현을 만드는 연산입니다. 입력은 먼저 임베딩(embedding)으로 벡터화되고 위치 인코딩(positional encoding)이 더해집니다. 그 벡터를 세 개의 학습된 행렬로 사영해 질의 $Q$, 열쇠 $K$, 값 $V$를 만들고, 핵심 공식 $\text{Attention}(Q,K,V)=\text{softmax}\!\left(\dfrac{QK^{\top}}{\sqrt{d_k}}\right)V$로 문맥 벡터를 얻습니다.

셀프 어텐션 파이프라인
① 임베딩+위치$ x_i = E(t_i) + p_i $
② 사영$ Q=XW^Q,\ K=XW^K,\ V=XW^V $
③ 점수$ S = QK^{\top}/\sqrt{d_k} $
④ 정규화·가중합$ \text{softmax}(S)\,V $
핵심 용어
질의 $Q$ "무엇을 찾는가" — 현재 토큰의 탐색 벡터
열쇠 $K$ "나는 무엇인가" — 각 토큰의 색인 벡터
값 $V$ "실제 내용" — 가중합될 정보 벡터
$d_k$ 열쇠·질의 차원 (스케일 인자 $\sqrt{d_k}$의 근거)
임베딩 $E$ 토큰 → 밀집 벡터 $\mathbb{R}^{d}$ 사상
위치 인코딩 $p_i$ 순서 정보 주입 (셀프 어텐션은 순서 무시)
핵심
어텐션은 내적(유사도) → softmax(확률) → 가중합의 세 단계일 뿐입니다. 순환(RNN)과 달리 모든 쌍을 한 번에 병렬로 계산해 긴 의존성도 일정한 경로 길이로 연결합니다.

Self-attention lets every token in a sequence distribute attention over the others to build a contextual representation. Inputs are first turned into vectors by an embedding, with a positional encoding added. Those vectors are projected by three learned matrices into query $Q$, key $K$, value $V$, and the core formula $\text{Attention}(Q,K,V)=\text{softmax}\!\left(\dfrac{QK^{\top}}{\sqrt{d_k}}\right)V$ yields the context vectors.

Self-attention pipeline
① embed + position$ x_i = E(t_i) + p_i $
② project$ Q=XW^Q,\ K=XW^K,\ V=XW^V $
③ scores$ S = QK^{\top}/\sqrt{d_k} $
④ normalize · weight$ \text{softmax}(S)\,V $
Key terms
Query $Q$ "what am I looking for" — the current token's search vector
Key $K$ "what am I" — each token's index vector
Value $V$ "the actual content" — the info vector to be summed
$d_k$ key/query dimension (basis for the scale factor $\sqrt{d_k}$)
Embedding $E$ maps token → dense vector in $\mathbb{R}^{d}$
Positional enc. $p_i$ injects order (self-attention is order-agnostic)
Key
Attention is just three steps: dot product (similarity) → softmax (probabilities) → weighted sum. Unlike recurrence (RNN) it computes all pairs in parallel at once, connecting long-range dependencies with a constant path length.

스케일드 닷프로덕트 어텐션의 유도

Deriving Scaled Dot-Product Attention

공통 관점: 어텐션은 "질의와 열쇠의 유사도를 확률로 바꾸어 값을 가중 평균하라"는 한 문장입니다. 유사도 척도로 내적(dot product)을 쓰고, 차원이 커질 때 값이 폭발하는 것을 막기 위해 $\sqrt{d_k}$로 스케일합니다.

Unifying view: attention is one sentence — "turn the similarity of query and key into probabilities and take a weighted average of the values." Similarity is measured by the dot product, and we scale by $\sqrt{d_k}$ to prevent the values from exploding as the dimension grows.

① 유사도 점수

내적 = 정렬 정도
🎯 질의 벡터와 열쇠 벡터가 같은 방향일수록 큰 점수
$ s_{ij} = q_i\!\cdot\! k_j = \sum_{m=1}^{d_k} q_{im}k_{jm} $

토큰 $i$의 질의와 토큰 $j$의 열쇠 사이 내적으로 "얼마나 관련되는가"를 잰다.

② $\sqrt{d_k}$ 스케일

분산 폭발 방지
📏 성분이 독립·단위분산이면 내적 분산 $\approx d_k$
$ \tilde{s}_{ij} = \dfrac{q_i\!\cdot\! k_j}{\sqrt{d_k}} $

$d_k$가 크면 내적이 커져 softmax가 포화(기울기 소실). $\sqrt{d_k}$로 나눠 분산을 $\approx 1$로 되돌린다.

③ softmax · 가중합

확률 → 값 평균
⚖️ 점수를 합이 1인 가중치로, 값 벡터를 섞는다
$ \alpha_{ij}=\dfrac{e^{\tilde{s}_{ij}}}{\sum_{j'}e^{\tilde{s}_{ij'}}},\ \ z_i=\sum_j \alpha_{ij}v_j $

softmax로 주의 분포 $\alpha$를 얻고, 값 $V$를 그 분포로 가중합해 문맥 벡터 $z_i$를 낸다.

① Similarity score

dot product = alignment
🎯 the more the query and key point the same way, the higher the score
$ s_{ij} = q_i\!\cdot\! k_j = \sum_{m=1}^{d_k} q_{im}k_{jm} $

The dot product between token $i$'s query and token $j$'s key measures "how relevant."

② $\sqrt{d_k}$ scaling

prevent variance blow-up
📏 with independent unit-variance entries the dot-product variance $\approx d_k$
$ \tilde{s}_{ij} = \dfrac{q_i\!\cdot\! k_j}{\sqrt{d_k}} $

Large $d_k$ makes dot products large, so softmax saturates (vanishing gradients). Dividing by $\sqrt{d_k}$ restores variance to $\approx 1$.

③ softmax · weighted sum

probabilities → value average
⚖️ turn scores into weights summing to 1, then blend the value vectors
$ \alpha_{ij}=\dfrac{e^{\tilde{s}_{ij}}}{\sum_{j'}e^{\tilde{s}_{ij'}}},\ \ z_i=\sum_j \alpha_{ij}v_j $

softmax gives the attention distribution $\alpha$; summing $V$ under it yields the context vector $z_i$.

주의 · 왜 하필 $\sqrt{d_k}$인가
$q,k$ 성분이 평균 0, 분산 1로 독립이라 가정하면 내적 $q\!\cdot\! k$의 분산은 $d_k$가 됩니다. 표준편차 $\sqrt{d_k}$로 나눠야 분산이 $\approx 1$로 정규화되어 softmax가 포화 없이 부드러운 기울기를 유지합니다. (출처: Vaswani et al. 2017, §3.2.1 각주)
Caution · why exactly $\sqrt{d_k}$
Assuming $q,k$ entries are independent with mean 0 and variance 1, the dot product $q\!\cdot\! k$ has variance $d_k$. Dividing by the standard deviation $\sqrt{d_k}$ normalizes the variance to $\approx 1$, keeping softmax unsaturated with smooth gradients. (Source: Vaswani et al. 2017, §3.2.1 footnote)

"Attention Is All You Need"와 순환의 종말

"Attention Is All You Need" and the End of Recurrence

2017년 Google Brain·Google Research의 Vaswani 외 8인은 순환(RNN)이나 합성곱 없이 어텐션만으로 시퀀스를 처리하는 트랜스포머를 제안했습니다. RNN은 토큰을 하나씩 순차 처리해 병렬화가 어렵고, 멀리 떨어진 토큰을 잇는 경로가 길어 학습이 힘들었습니다. 셀프 어텐션은 모든 토큰 쌍을 동시에 연결해 이 두 문제를 한꺼번에 풉니다.

긴 의존성의 경로 길이를 $O(n)$에서 $O(1)$로 줄인 것이 트랜스포머의 핵심 이점이다.

논문은 멀티헤드 어텐션(여러 사영으로 서로 다른 관계를 병렬 학습)과 사인·코사인 위치 인코딩을 도입했습니다. 결과 아키텍처는 기계번역에서 당시 최고 성능(WMT 2014 EN-DE에서 28.4 BLEU — 논문 보고값)을 훨씬 적은 학습 비용으로 달성했고, 이후 BERT·GPT 등 거의 모든 대규모 언어모델의 뼈대가 되었습니다.

꼭 알아야 할 점
  • 트랜스포머는 순환·합성곱 없이 어텐션만으로 시퀀스를 처리한다.
  • 병렬화 + 짧은 경로 길이($O(1)$)가 핵심 이점.
  • 위치 인코딩이 없으면 어텐션은 순서를 모른다.

참고: Vaswani et al. (2017) NeurIPS, arXiv:1706.03762 / BLEU 값은 논문 Table 2 보고값

In 2017, Vaswani and 7 co-authors at Google Brain / Google Research proposed the Transformer, which processes sequences using attention alone, without recurrence or convolution. RNNs process tokens one by one, are hard to parallelize, and have long paths connecting distant tokens. Self-attention connects all token pairs at once, solving both problems together.

Shrinking the path length for long-range dependencies from $O(n)$ to $O(1)$ is the Transformer's core advantage.

The paper introduced multi-head attention (learning different relations in parallel via several projections) and sinusoidal positional encoding. The resulting architecture reached state-of-the-art machine translation (28.4 BLEU on WMT 2014 EN-DE — value reported in the paper) at far lower training cost, and became the backbone of nearly every large language model since, including BERT and GPT.

Must know
  • The Transformer processes sequences with attention alone, no recurrence/convolution.
  • Parallelism + short path length ($O(1)$) are the core advantages.
  • Without positional encoding, attention knows no order.

2-Token Attention — Play with softmax and the $\sqrt{d_k}$ Scale

두 토큰 A·B에 대해 질의 토큰이 계산한 원시 내적 점수 $s_A,s_B$와 차원 $d_k$를 조절해보세요. 스케일된 점수를 softmax에 넣으면 토큰 A에 주는 주의 확률 $\alpha_A$가 나옵니다: $\alpha_A=\dfrac{e^{s_A/\sqrt{d_k}}}{e^{s_A/\sqrt{d_k}}+e^{s_B/\sqrt{d_k}}}$. $d_k$가 커질수록 분포가 어떻게 부드러워지는지 관찰하세요.

For two tokens A and B, adjust the raw dot-product scores $s_A,s_B$ the query token computed, and the dimension $d_k$. Feeding the scaled scores to softmax gives the attention probability $\alpha_A$ on token A: $\alpha_A=\dfrac{e^{s_A/\sqrt{d_k}}}{e^{s_A/\sqrt{d_k}}+e^{s_B/\sqrt{d_k}}}$. Watch how a larger $d_k$ makes the distribution softer.

공학적 감각 — 주의 확률 $\alpha\in[0,1]$이고 한 행의 합은 항상 1이다. 점수 차 $s_A-s_B$가 클수록 $\alpha_A\to1$(날카로운 주의), 차가 0이면 $\alpha_A=0.5$(균등). 스케일 인자는 실효 점수 차 = (원시 차)/$\sqrt{d_k}$로 작동한다: 논문 기본 $d_k=64$면 $\sqrt{d_k}=8$이라 원시 차 8이 실효 차 1로 줄어든다. 큰 $d_k$에서 스케일을 빼면 점수가 폭발해 softmax가 한 토큰에 몰빵(포화)하고 기울기가 사라진다. (출처: Vaswani et al. 2017, §3.2.1)
Engineering sense — attention probability $\alpha\in[0,1]$ and each row always sums to 1. A larger score gap $s_A-s_B$ pushes $\alpha_A\to1$ (sharp attention); a zero gap gives $\alpha_A=0.5$ (uniform). The scale factor acts as an effective gap = (raw gap)/$\sqrt{d_k}$: with the paper's default $d_k=64$, $\sqrt{d_k}=8$, so a raw gap of 8 shrinks to an effective gap of 1. Removing the scale at large $d_k$ lets scores explode, softmax saturates onto one token, and gradients vanish. (Source: Vaswani et al. 2017, §3.2.1)

🎲

🎲
Intuition — an LLM does not pick "the next word" in one shot. It assigns a score (logit) to every candidate in the vocabulary, turns those scores into a probability distribution, and then samples one. Lowering the temperature concentrates mass on the top candidate (more deterministic); raising it flattens the distribution (more diverse). The knobs a prompt engineer actually touches live in this normalization–sampling stage. (Source: Holtzman et al. 2020, "The Curious Case of Neural Text Degeneration", ICLR)

The final layer of a language model emits a logit vector $z\in\mathbb{R}^V$ over a vocabulary of size $V$. Logits are unnormalized scores — any real value. Normalizing them with softmax yields probabilities $P_i=\dfrac{e^{z_i/T}}{\sum_j e^{z_j/T}}$, guaranteeing $\sum_i P_i=1$. The temperature $T$ tunes the peakedness of the distribution — as $T\to 0$ it becomes argmax (greedy), and as $T\to\infty$ it approaches the uniform distribution.

Softmax and Temperature — Why the Distribution Sharpens

공통 관점: 온도가 있는 소프트맥스 $P_i=\dfrac{e^{z_i/T}}{\sum_j e^{z_j/T}}$는 로짓 차이만 의미를 가집니다 — 모든 로짓에 같은 상수를 더해도 $P$는 불변(이동 불변성). 결정하는 것은 $ (z_i-z_j)/T $, 곧 온도로 나눈 로짓 간 격차입니다.

Unifying view: tempered softmax $P_i=\dfrac{e^{z_i/T}}{\sum_j e^{z_j/T}}$ depends only on logit differences — adding the same constant to all logits leaves $P$ unchanged (shift invariance). What matters is $ (z_i-z_j)/T $, the logit gaps scaled by temperature.

핵 샘플링(top-p)은 왜 등장했는가

Why Nucleus Sampling (top-p) Emerged

2020년 Holtzman, Buys, Du, Forbes, Choi는 "신경 텍스트 퇴화(neural text degeneration)"를 지적했습니다. 그리디·빔서치처럼 확률 최대만 좇으면 문장이 반복 루프에 빠지고, 반대로 순수 온도 샘플링은 확률이 낮은 꼬리 토큰을 물어 비문·헛소리가 됩니다.

사람의 언어는 "가장 그럴듯한 단어"의 나열이 아니다.

그들의 해법이 핵 샘플링(top-p)입니다: 확률을 내림차순으로 더해 누적이 $p$(예: 0.9)를 넘는 최소 후보집합("핵")만 남기고 나머지 꼬리를 잘라 재정규화한 뒤 그 안에서 뽑습니다. top-k가 후보 개수를 고정하는 데 비해, top-p는 확률 질량을 고정하므로 분포가 뾰족할 땐 후보가 적고 평평할 땐 많아 맥락에 적응합니다.

꼭 알아야 할 점
  • top-p는 누적확률, top-k는 후보 개수를 자른다.
  • 온도는 모양을, top-p/k는 후보집합을 바꾼다 — 직교적 손잡이.
  • 정확성엔 낮은 $T$, 창의성엔 top-p(예: 0.9~0.95).

참고: Holtzman et al. (2020) ICLR / Fan et al. (2018) top-k / Ackley et al. (1985) 온도 소프트맥스

In 2020, Holtzman, Buys, Du, Forbes, and Choi diagnosed "neural text degeneration." Chasing only the most probable tokens (greedy/beam search) traps text in repetition loops; conversely, pure temperature sampling bites low-probability tail tokens, producing incoherent nonsense.

Human language is not a string of "most-likely words."

Their fix is nucleus sampling (top-p): sum probabilities in descending order, keep the smallest candidate set ("nucleus") whose cumulative mass exceeds $p$ (e.g. 0.9), cut the rest of the tail, renormalize, and sample within it. Where top-k fixes the number of candidates, top-p fixes the probability mass, so it is context-adaptive — few candidates when the distribution is sharp, many when it is flat.

Must know
  • top-p cuts by cumulative probability; top-k cuts by candidate count.

Q1.As temperature $T$ is lowered toward 0, what does softmax do?

Explanation Answer ② — since $P_i/P_j=e^{(z_i-z_j)/T}$, as $T{\to}0$ the gaps blow up and all mass concentrates on the max-logit token (greedy). Wrong ①·③: ① uniform is the $T{\to}\infty$ limit; ③ softmax always guarantees $\sum P_i=1$. class n=41 · ① 6 · ② 31 · ③ 4

Q2.top-p(핵 샘플링)와 top-k의 차이로 옳은 것은?

해설 정답 ② — top-p는 누적확률이 $p$를 넘는 최소 후보집합("핵")을 남기므로 후보 수가 맥락에 따라 변하고, top-k는 상위 $k$개로 개수를 고정한다(Holtzman et al. 2020). 오답 ①·③: ①개수 고정은 top-k만 해당, ③둘 다 온도가 아니라 후보집합을 절단하는 손잡이다. 학급 n=41 · ① 5 · ② 29 · ③ 7

Explanation Answer ② — top-p keeps the smallest set ("nucleus") whose cumulative mass exceeds $p$, so its size adapts to context, whereas top-k fixes the count at the top $k$ (Holtzman et al. 2020). 핵심 정리 · 자주 하는 실수 · 시험 포인트

Key Points · Common Mistakes · Exam Focus

핵심 정리

  • 로짓 $z_i$ → 소프트맥스 → 확률 $P_i=\dfrac{e^{z_i/T}}{\sum_j e^{z_j/T}}$, $\sum_i P_i=1$.
  • 온도: $T{\downarrow}$ 결정적(argmax), $T{\uparrow}$ 다양(균등). 결정하는 건 $(z_i-z_j)/T$.
  • 절단: top-p=누적확률 질량, top-k=후보 개수. 온도(모양)와 직교.

⚠ 자주 하는 실수

  • 온도가 로짓의 절대값을 바꾼다고 오해 → 실제는 격차만 의미.
  • top-p와 top-k를 혼동 → 질량 vs 개수.
  • 정확성 작업에 높은 $T$ 사용 → 코드·수식은 낮은 $T$.

시험 포인트

  • 주어진 로짓·$T$로 $P_i$ 직접 계산 (분모=정규화상수).
  • $T{\to}0$·$T{\to}\infty$ 두 극한의 의미(그리디 vs 균등).
  • top-p 핵 크기 판별 및 top-k와의 차이(Holtzman et al. 2020).

Key points

  • Logit $z_i$ → softmax → probability $P_i=\dfrac{e^{z_i/T}}{\sum_j e^{z_j/T}}$, with $\sum_i P_i=1$.
  • Temperature: $T{\downarrow}$ deterministic (argmax), $T{\uparrow}$ diverse (uniform). What matters is $(z_i-z_j)/T$.
  • Truncation: top-p=cumulative mass, top-k=candidate count. Orthogonal to temperature (shape).

⚠ Common mistakes

  • Thinking temperature changes the absolute logits → only the gaps matter.
  • Confusing top-p with top-k → mass vs count.
  • Using high $T$ for accuracy tasks → code and math need low $T$.

Exam focus

  • Compute $P_i$ from given logits and $T$ (denominator = partition function).
  • Meaning of the two limits $T{\to}0$ / $T{\to}\infty$ (greedy vs uniform).
  • Determine nucleus size for top-p and its difference from top-k (Holtzman et al. 2020).

컨텍스트 윈도우 — 모델의 '작업 기억'

The Context Window — the Model's "Working Memory"

📖
직관 — 책상 위에 펼칠 수 있는 종이 장수가 정해져 있다고 생각하세요. 새 종이를 올리면 오래된 종이는 밀려 떨어집니다. 게다가 사람이 긴 문서를 읽을 때 앞머리와 끝은 잘 기억하지만 한가운데는 흐려지듯, LLM도 정답이 컨텍스트 중간에 있으면 정확도가 뚝 떨어집니다. (출처: Liu et al. 2023 "Lost in the Middle", TACL 2024)
📖
Intuition — Imagine a desk that fits only a fixed number of pages at once. Add a new page and the oldest one slides off. And just as people reading a long document remember the beginning and the end but blur on the middle, an LLM's accuracy drops sharply when the answer sits in the middle of the context. (Source: Liu et al. 2023 "Lost in the Middle", TACL 2024)

한 번의 추론에서 모델이 볼 수 있는 토큰의 최대 개수가 컨텍스트 윈도우(context window)입니다. 프롬프트, 시스템 지시, 검색으로 붙인 문서, 그리고 모델이 생성할 출력까지 모두 이 예산 $L$ 안에 들어가야 합니다. $L$을 넘는 토큰은 잘리거나 밀려나(truncation) 모델의 시야에서 사라집니다. 게다가 윈도우 안에 있더라도 위치(position)에 따라 활용도가 다릅니다 — 앞·뒤는 강하고 가운데는 약한 U자형 곡선을 보입니다.

The context window is the maximum number of tokens the model can see in a single inference. The prompt, system instructions, retrieved documents, and the output the model will generate must all fit inside this budget $L$. Tokens beyond $L$ are cut or pushed out (truncation) and vanish from the model's view. And even inside the window, utility varies with position — a U-shaped curve that is strong at the ends and weak in the middle.

왜 U자 곡선인가 — 위치가 attention을 흔든다

Why the U-Curve — Position Skews Attention

공통 관점: 트랜스포머는 각 토큰이 다른 토큰에 얼마나 주목할지를 attention 가중치로 정합니다. 이 가중치는 위치 인코딩과 학습 편향 때문에 문장의 처음·끝에 쏠리는 경향이 있고, 그 결과 중간 위치의 근거는 상대적으로 희석됩니다.

Unifying view: a transformer decides how much each token attends to others via attention weights. Because of positional encoding and training bias, these weights tend to concentrate at the start and end, so evidence in the middle is comparatively diluted.

2023년 Nelson Liu 외 (Stanford)는 다문서 질의응답과 key-value 검색 과제에서, 정답이 담긴 문서를 입력 안 여러 위치로 옮겨가며 모델 성능을 측정했습니다. 결과는 인상적이었습니다 — 정답이 맨 앞이나 맨 뒤에 있을 때 정확도가 가장 높고, 한가운데로 갈수록 성능이 뚜렷하게 떨어지는 U자 곡선이 여러 모델에서 반복됐습니다.

모델은 컨텍스트 중간의 관련 정보를 제대로 쓰지 못한다.

주목할 점은, 윈도우를 키워 더 긴 입력을 넣을 수 있게 해도 중간 소실 자체는 사라지지 않았다는 것입니다. 즉 "더 긴 컨텍스트 = 더 좋은 활용"이 아니며, 무엇을 어디에 배치하느냐(context engineering)가 성능을 좌우합니다. 실무 처방은 명확합니다 — 핵심 근거·지시는 처음이나 끝에 두고, RAG에서는 재순위화(re-ranking)로 가장 관련 높은 문서를 양끝으로 보내라.

꼭 알아야 할 점
  • 정확도는 위치의 함수 — 양끝 강, 중간 약(U자).
  • 윈도우 확대 ≠ 중간 소실 해결.
  • 중요 정보는 앞/뒤, RAG는 re-ranking.

참고: Liu, Lin, Hewitt, Paranjape, Bevilacqua, Petroni, Liang (2023) "Lost in the Middle: How Language Models Use Long Contexts", arXiv:2307.03172 / TACL 2024.

In 2023 Nelson Liu et al. (Stanford) measured model performance on multi-document QA and key-value retrieval while moving the document containing the answer to different positions in the input. The result was striking — accuracy was highest when the answer sat at the very start or end, and dropped markedly toward the middle: a U-shaped curve reproduced across several models.

Models fail to use relevant information located in the middle of the context.

Notably, enlarging the window so longer inputs fit did not remove the middle loss itself. So "longer context = better use" is false; instead, what you place where (context engineering) governs performance. The practical prescription is clear — put key evidence and instructions at the start or end, and in RAG use re-ranking to push the most relevant documents to the ends.

Must know
  • Accuracy is a function of position — strong at ends, weak in middle (U).
  • Larger window ≠ fixing the middle loss.
  • Put key info at start/end; use re-ranking in RAG.

Ref: Liu, Lin, Hewitt, Paranjape, Bevilacqua, Petroni, Liang (2023) "Lost in the Middle: How Language Models Use Long Contexts", arXiv:2307.03172 / TACL 2024.

핵심 정리

  • 컨텍스트 윈도우 $L$ = 한 번에 볼 수 있는 최대 토큰 수; 입력+출력이 모두 $L$ 안이어야 함.

⚠ 자주 하는 실수

  • 윈도우만 키우면 다 해결된다는 오해 → 중간 소실은 남는다.
  • 핵심 정보를 문맥 정중앙에 파묻기.
  • 출력 토큰 예산을 잊어 답변이 잘림.

시험 포인트

  • 토큰 예산식 $\sum n_i \le L$로 $n_{out}$ 구하기.
  • U자 모델 $\text{acc}(p)=a+b(2p-1)^2$에 위치 대입 계산.
  • Lost in the Middle의 의미와 배치·재순위화 처방(Liu et al. 2023).

Key points

  • Context window $L$ = max tokens seen at once; input + output must fit within $L$.
  • Accuracy is a function of position — high at ends, lowest at center: the U-curve (Lost in the Middle).
  • Put key instructions & evidence at the start/end; in RAG use re-ranking to the ends.

⚠ Common mistakes

  • Believing a bigger window fixes everything → the middle loss remains.
  • Burying key information in the exact center.
  • Forgetting the output-token budget → the answer gets truncated.

Exam focus

  • Plug position into the U-model $\text{acc}(p)=a+b(2p-1)^2$.
